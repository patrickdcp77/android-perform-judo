# Development

- Toute décision structurante doit avoir un ADR (1 fichier).
- Chaque procédure doit être **reproductible** (commande exactes, prérequis, résultats attendus).
## Règles

- `development/protocols/` : protocoles capteur (BLE, formats paquets, versions)
- `development/checklists/` : listes de vérification (avant commit, avant release)
- `development/decisions/` : ADR (Architecture Decision Records)
- `development/procedures/` : pas-à-pas reproductibles (setup, build, debug, releases)
## Structure

Ce dossier sert de "mémoire de projet" : procédures, conventions, décisions d’architecture, checklists.
